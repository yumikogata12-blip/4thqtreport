        <p>
            In this lesson, I learned that where we place our JavaScript code is just as important as the code itself. Placing scripts in the <span class="highlight">HEAD</span> section ensures they are pre-loaded before any user interaction, making it perfect for function calls. Alternatively, placing scripts in the <span class="highlight">BODY</span> allows them to execute as the page loads. I also discovered the efficiency of <span class="highlight">External JavaScript</span>, where we can save code in a .js file and link it to multiple HTML pages, which saves so much time and keeps the project organized.
        </p>

        <p>
            The lesson also emphasized the importance of commenting—using <span class="highlight">//</span> for single lines or <span class="highlight">/* ... */</span> for blocks—which is essential for debugging and explaining logic to others. It is a vital practice for keeping code maintainable and clear, especially when working on larger, more complex websites.
        </p>

        <p>
            Finally, I learned the critical guidelines for coding in JavaScript. JavaScript is strictly <span class="highlight">Case Sensitive</span>, meaning variable names like "myHome" and "MyHome" are completely different entities. While JavaScript is forgiving regarding <span class="highlight">White Space</span>, using it effectively makes the code much easier to read. I also learned that while we can break lines of code using a backslash, we must be careful never to break commands mid-way, as this would break the functionality. Following these "Do's and Don'ts" is a great way to avoid unnecessary errors and become a more disciplined programmer.
        </p>
